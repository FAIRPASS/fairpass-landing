## 1. 전체 흐름 요약

```
[랜딩페이지 견적폼 입력]
    ↓ (자동 계산 + 요약 표시)
[견적 요청하기 버튼] → 이메일/전화 입력 모달
    ↓ POST /api/quote-email
[Supabase DB 저장 + 고객에게 자동 수신확인 메일]
    ↓
[관리자페이지 견적문의 탭]
    ↓ 📄 보기 클릭 → 견적 발송 모달
[비용 내역 수정 + 단가 조정 + 할인 + 메모]
    ↓ 고객에게 발송하기 → POST /api/quote-send
[고객 이메일로 최종 견적서 발송 + DB content_edited 저장]
    ↓ 또는
[PDF 미리보기 버튼 → 브라우저 새 창에서 인쇄/PDF 저장]
```

### II. 종이명찰 (기본값: 일반형 중)
- 기본 단가: ₩1,500/장
- 수량 = 참가자 수와 동일

### III. 목걸이줄 (기본값)
- 기본 단가: ₩500/개
- 수량 = 참가자 수와 동일

### IV. 무인발권키오스크
- 단가: ₩100,000/대/일
- 키오스크 권장 대수: 별도 추천 로직 (attendees, days 기반)

### V. 발송 서비스 (관리자페이지 선택 추가)
- 알림톡: ₩20/건
- 문자(LMS): ₩100/건
- 이메일: ₩50/건

### VI. 운송비 (지역별, 왕복 기준)

|------|--------|

| 홍성/홍천 | ₩500,000 |
| 속초/전주 | ₩600,000 |

| 부산/울산/창원/경주/무안 | ₩800,000 |
| 제주 | ₩1,200,000 |

### VII. 설치/철거 매니저 인건비
- 1일: ₩300,000

### VIII. 현장 상주 매니저 인건비 (선택)

### 부가가치세

## 6. API 구조

### POST /api/quote-email
- **역할**: 고객이 견적 요청하기 버튼 클릭 시 호출
- **저장**: Supabase inquiries 테이블에 content_original + 기본 정보 저장
- **발송**: 고객에게 수신확인 자동 이메일 발송 (Resend)
- **FROM**: noreply@fairpass.world (발송 전용)

### POST /api/quote-send
- **역할**: 관리자가 고객에게 견적서 이메일 발송 시 호출
- **Body**: `{ password, id, customerEmail, customerName, quoteContent, quoteTotal, quoteDiscount, quoteNotes }`
- **발송**: Resend로 HTML 이메일 발송 (CC: fairpass@fairpass.world)
- **저장**: DB의 content_edited, total, quote_sent_at 업데이트
- **parseQuoteContent()**: quoteContent에서 담당자/행사정보/비용상세/총계 파싱

## 7. 관리자페이지 견적문의 탭 기능 (admin/index.html)

### 목록 테이블
- 담당자, 소속, 이메일, 전화, 참가자수, 일수, 총계, 상태 표시
- 📄 보기 버튼 → 견적 발송 모달 열림
- 발송 상태 배지 (✓ 발송 / 미발송)

### 견적 발송 모달 (openQuoteSendModal)
- 로드 시: `content_edited || content` 우선 사용
- `parseQuoteContent()` → 비용 상세 섹션만 파싱하여 편집 테이블 렌더링

#### ⚙️ 세부 옵션 (관리자 전용 선택)
| 옵션 | 설명 |
|------|------|
| 종이명찰 타입 | 일반형(중) ₩1,500 / 일반형(대) ₩2,000 / 티켓형 ₩1,500 / 가슴명찰형 ₩1,500 |
| 목걸이줄 타입 | AI(P) ₩1,300 / B1(P) ₩600 / A1 ₩1,000 / A2 ₩1,100 / B1 ₩500 / B2 ₩600 |
| V. 발송 서비스 | 알림톡 ₩20 / 문자LMS ₩100 / 이메일 ₩50 (체크 시 키오스크 다음에 삽입) |
| VIII. 현장 상주 매니저 | ₩300,000/인/일, 인원 수 입력 (체크 시 설치/철거 다음에 삽입) |

- 선택 시 → 빌링 테이블에 해당 행 자동 삽입/업데이트
- 선택 해제 시 → 해당 행 자동 제거

#### 비용 내역 편집 테이블
- 항목명(좌)·금액(우) 모두 **직접 클릭하여 수정 가능** (text input)
- + 항목: 새 빈 행 추가
- 🧮 합계: 모든 금액 행 합산 → 총 금액 자동 입력
- 💾 저장: DB content_edited에 저장 (발송 없이 저장만)
- 수정 내용은 `syncBillingToContent()` → `qs_content` textarea에 실시간 반영

#### 할인
- ₩(금액) 또는 %(비율) 선택
- 입력 시 최종 금액 자동 재계산

#### 발송
- 총 금액 필수 입력
- "고객에게 발송하기" → POST /api/quote-send
- 발송 완료 시 DB 업데이트 + 목록 새로고침

### PDF 미리보기 (openQuotePdf)
- 브라우저 새 창에서 HTML 형식 견적서 열기
- `content_edited || content` 데이터 사용
- 인쇄 버튼 → 브라우저 인쇄 기능으로 PDF 저장
- 포함 내용: FAIRPASS 로고, 신청자 정보, 행사 정보(지역/기간/설치철거), 비용 내역, 정산 문구, 총계, 도입 안내서

## 10. 설계 원칙 & 주의사항

- **랜딩페이지**: 심플하게 유지. 명찰 타입·줄 타입·발송 서비스·현장 매니저 세부 선택은 **관리자 전용**
- **content vs content_edited**: `content`(원본)는 절대 덮어쓰지 않음. 관리자 수정분은 `content_edited`에만 저장
- **발송 서비스 삽입 위치**: IV(키오스크) 다음, VI(운송비) 이전 (V번 순서)
- **현장 상주 매니저 삽입 위치**: VII(설치/철거) 다음, IX(출장비) 이전 (VIII번 순서)
- **Flatpickr**: 행사 기간은 range 모드, 설치/철거는 single 모드. 값은 hidden input으로 읽음
- **로고 이메일**: `https://fairpass.world/images/FAIRPASS%28W%2BBlack%29.png` (이메일 클라이언트 이미지 차단 시 텍스트 "FAIRPASS" 대체)
- **force push 금지**: origin(blueorigin2021)에 절대 force push 금지 (어드민 저장 데이터 손실 전례)
